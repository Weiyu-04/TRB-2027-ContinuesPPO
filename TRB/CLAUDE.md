## §0 项目一句话

可证明 COLREGs（国际海上避碰规则）**方向**合规此前要么牺牲连续控制（Krasowski & Althoff 2024 用**离散**动作 + action masking），要么牺牲合规保证（连续动作工作全退回**软奖励**）。本文把**投影式安全盾**首次落到海事 COLREGs：

```
观测 → SAC → 连续动作 u → ① 状态机判相遇态势+合规方向
                          → ② zonotope 安全动作集（合规方向 + 无碰撞）
                          → ③ 二次规划投影修正 → 安全动作 → 环境
```

**卖点 = 三重交集**：连续动作 ∩ 可证明**方向**合规 ∩ COLREGs（此前空白）。
> **"provably" 诚实档（`03` L164 审计 + L165 gap#1 攻坚后·正式命题见 `Paper/可证明层_正式命题_0707.md`）**：能严证 = **远场单步无碰(命题1·v_obs≤9.5 全池 2000 场景验 max 7.10)+每让路步方向合规(命题2·give-way 触发 ~13-16% 相遇[encounter 级·非 per-step])+紧急迫近不可避 sound 证书(命题3·gap#1)**；**🚫 绝不写 provably collision-free / provably compliant(裸) / 完整可证明覆盖**（紧急态 55-60% 是经验 Alg.1·多障碍/机动他船未覆盖·全押 CV+单障碍须显式假设）。可证明档 = **中**。

## §2 工作纪律（铁律）

- **🔴 对 user 一律说人话（user 2026-07-09 明令·硬要求）**：所有**对 user 的分析 / 汇报 / 聊天 / 沟通**必须**通俗中文**，让 user 一眼看懂为准——**别堆英文缩写和术语**（IQM、checkpoint、replay、bit-identical、config 这类），能用大白话就用大白话（多用生活类比、短句、少嵌套）；**必须用的专有名要顺带一句人话解释**。密集 house style + 英文术语**只准留在代码注释和 `02`/`03` 笔记里**（那是给下个 AI 窗口看的），**绝不倒给 user**。违反 = 没读懂 user 需求。参见 memory [[user-prefers-plain-language-explanations]]。
- **中文严格**：英文术语必译。专有名 / 算法名 / 库名保留（zonotope / SAC / COLREGs / CommonOcean / 二次规划），**首次出现带简短中文注解**。
- **PI 严谨**：施工参数必须 **fact-based**（亲自读论文 / 读代码核实，不靠转述、不靠记忆）；定量声明配 hedge（概率估计）；不编数字；不 cherry-pick。
- **节点复查**：每完成一个里程碑停下自查，**不在一个回复里 chain 多个 tool call 累积错误**。
- **通过门强制**：门没过不往下走（见 `01`）。这是"踏实"的落地形式。
- **基础设施代码 = 最高审核标准**（user 2026-06-09 强调"重中之重、不许出错"）：每件基础设施模块（环境 / 动力学 / 观测 / 奖励 / 状态机 / 投影 等）必须 **fact-based 对论文或官方代码 → 写冒烟测试 → 派 ≥2 个独立 agent 多轮对抗复核 → 措辞收紧到诚实 → 过了才进下一件**。`代码/trb_env/usv_dynamics.py` 已立此范式（见 `02` 已完成）。
- **⚠️ "代码跑通/测试绿 ≠ 没问题"（user 2026-06-17 铁律）**：审核必须往深做，综合核 ① **参数设计合理性**（每个常量/阈值/默认值的依据与边界）② **预见一切可能问题、把防御性设计做在前面**（不是出了再补）③ **跨模块联动的潜在问题**（与其他件接线时的时序/口径/状态泄漏/契约）④ **代码逻辑严密性** ⑤ **数学严密性**——不止"能跑/测试过"。详 `03` D39。
- **工程性能优化（编写时即做，user 2026-06-09 D17 + 2026-06-17 重申）**：训练吃 CPU、**随时想"让计算更快"**——热路径用高效计算逻辑（numpy 向量化 / 解析替 shapely、缓存复用、预加载、VecEnv、**先实测瓶颈再优化** L40），**任何优化必等价验证、不牺牲正确性 / 论文忠实度**（见 memory `code-perf-optimize-while-writing`）。
- **不擅自实现**：讨论 / 探索阶段只给方案 + 最推荐，**等 user 明确"去做"再动代码**（尤其会训练、会烧 GPU 的）。
- **Tier 决策分级**：局部可逆 → 直接做 + 简短汇报；影响方法 / 流程 / 跨文件 → 给推荐 + 等 user 确认；不可逆 / 烧 GPU 长跑 / 投稿决策 → **必等 user 拍板，不假设授权**。
