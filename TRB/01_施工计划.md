## Phase 0 — 立项与地基（消除不确定性）

**目标**：把蓝图里"AI 多轮检查"得到的施工事实，换成亲自读论文 / 读代码核实过的 ground truth。方向对，但施工参数不能靠转述。

**关键任务**
- 深读 **Krasowski 2024**（`参考资料/2402.08502v2.pdf`，锚点论文），提取 5 样：
  ① CommonOcean 动力学（确认偏航受限模型 + 连续控制 `u=[a, ω]`）
  ② COLREGs 状态机形式化（相遇态势分类扇区角、`Δ_large_turn`、`Δ_no_turn`、CPA / `t_horizon`）
  ③ 违规率 / 碰撞率 / 到达率的精确计数口径

- 深读 **Markgraf 2026**（`参考资料/2509.12833v2.pdf`）：serl-sprl 开源仓库结构、SE-RL（安全盾当环境一部分）接线、zonotope 投影 / 二次规划工程实现（附录 A.1）、action aliasing（动作混叠）惩罚。
- 摸清 **CommonOcean**：能否 pip 安装、gym 接口、场景格式、录制的 AIS（船舶自动识别系统）场景。
- 摸清 **serl-sprl**：能否克隆跑通、许可证、依赖。

## Phase 1 — 环境可行性 + 基线锚点（最高优先级风险）

**目标**：确认"环境对、锚点在"。

**关键任务**：跑通 CommonOcean 单他船对遇场景 → **复现 Krasowski 的 Discrete-safe 基线** → 锁死违规率判定口径（全取 Krasowski 同值，落参数表）→ 跑通 Baseline（无合规）+ Rule-reward（软奖励）两个连续基线。

**通过门 1**：Discrete-safe 复现出的三项指标与 Krasowski Table III **数量级一致**（不要求完全相同，要求"锚点在"）。**复现失败 → 停下诊断，不进 Phase 2。**

---

## Phase 2 — 投影最小闭环（档位 A，可达性主场）

**目标**：验证方案最核心假设——连续动作能被投影进"合规 + 无碰撞"集合。

**关键任务**：单他船 + 对遇一种态势 → `U_colregs: ω ≤ -ω_turn`（右转）→ `U_collision-free`：单前瞻时刻 + 分离超平面 + 标量一阶展开 + 大安全裕度（蓝图 §12.3.3 修订版）→ 路线 1 的二次规划投影（cvxpy / OSQP 直接在多胞形上投影，先不显式转 zonotope）→ 用**非 SAC 的简单策略**先验证投影逻辑本身。

**通过门 2**：简单策略下，对遇场景投影后整段轨迹零违规、零碰撞、能到目标。**这门一过 = 方案核心假设成立。**

---

## Phase 3 — 接 SAC + 全场景 + 全基线对比

**目标**：跑出 Continuous-safe，产第一版钱图。⚠️ 这里开始大量训练，与 T-ITS 抢卡 → 错峰。

**关键任务**：SE-RL 方式接 SAC（主体不改）→ 扩到交叉（左/右）、追越、直航全态势 → 多前瞻时刻 → action aliasing 惩罚项（与投影修正量成正比）→ 四方对比（Baseline / Rule-reward / Discrete-safe / Continuous-safe）同口径。

**通过门 3**：Continuous-safe 全态势零违规零碰撞，且效率指标（到达率 / 轨迹长度 / 平滑度）不输甚至优于 Discrete-safe（钱图核心结论：连续投影把离散 masking 掉的 4–5% 到达率代价压低）。

---

## Phase 4 — 档位 B：不变集终端约束（provably 开关，最不确定）

**目标**：一步前瞻 ≠ 可证明安全（需递归可行性 / 前向不变性）。这是蓝图反复强调的"最重要待定开关"。

**关键任务**：预计算鲁棒控制不变集 / reach-avoid（可达-规避）集 → 把"落点仍在不变集内"做进投影硬约束（蓝图 §12.5 应对 (i)，把保证拉回静态投影层面，缓解 SAC × 档位 B 的理论张力）→ 诚实边界照抄 Limitations。

**通过门 4**：不变集能稳定算出、终端约束进投影、不破坏 Phase 3 结果。
- **稳** → 标题用 "provably COLREGs-compliant ... in continuous action space"。
- **不稳** → 老实退回档位 A 措辞 "continuous-action COLREGs-compliant collision avoidance via projection-based action filtering"（去 provably），仍是有效贡献。

---

## Phase 5 — 紧急冲突场景 + 兜底验证

**目标**：让 `P=∅` 兜底逻辑真正被触发，不当纸上谈兵（蓝图 §12.4 同行复审指出的洞）。

**关键任务**：人工构造紧急冲突场景（两船夹击 / 他船不守规突然逼近）→ 触发 `P=∅` → 验证兜底（切换碰撞风险最小化、放松 COLREGs 方向约束）。把"双他船"从可选扩展提升为"至少有触发兜底的冲突算例"。

**通过门 5**：兜底被真实触发且行为正确。

---

## Phase 6 — 多种子 + 统计 + 写作（复用 USV 方法论）

**关键任务**：≥5 种子重跑（碰撞率 / 违规率 / 到达率的均值与方差）→ 配对检验 + 自助法置信区间 → 钱图定稿 → 写作（**7500 词、无附录**，完整数学留外部技术报告给链接）→ 投稿前再扫一轮 arXiv 确认"首次"。

**通过门 6**：核心结论在多种子下稳健（均值 + 方差），文稿符合 TRB 硬约束（见原始蓝图第 8 节）。
