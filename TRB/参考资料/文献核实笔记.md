## ③ 违规/碰撞/到达率计数口径（PAGE 12, §VII-A）— ✅ 与蓝图 §14 描述一致

评估指标：goal-reaching rate（到达率）、reward、episode length、collisions（碰撞）、emergency 使用率、rule violations（违规）。
**违规计数（精确口径，蓝图要对齐的就是这个）**：
- 直航态势：**每一个违反直航位置的时间步**计一次；
- give-way（交叉/追越/对遇）：**每一个该让路却未采取合规避让机动的相遇**计一次。

**⭐ step3(c) fact-check 深化（2026-06-10，亲读 2024 §VII-A p12 逐字 + commonocean-rules R_G3-6 MTL 映射，GPL v3 只读参照 D10）**：

2024 §VII-A 逐字："Rule violations reflect how often **per episode** the regular collision avoidance rules are violated. For that, we count: • every time step of violating the stand-on vessel position results; • every crossing, overtaking and head-on encounter for which no proper collision avoidance maneuver is taken." → Table III "Rules violated" = 平均每 episode 违规次数（Base handcrafted **2.65** / RR 2.24 / Safe **0**；recorded Base 0.75 / Safe 0）。

权威映射 commonocean-rules R_G3-6 MTL（Krasowski 2021，2024 复用同套规则）：
- **R_G6 stand-on（time-step 级）**：`G(keep → (no_turning U ¬keep))`。`no_turning` = keep onset 起**累积航向 |Δθ_cum| < Δno_turn=10°**（max_orientation_diff_no_change）。违反 = keep 期间累积 ≥10° 的**每个时间步** +1。
- **R_G3-5 give-way（encounter 级）**：`G((¬X ∧ G[1,6]X) → (F[0,12](maneuver_X) ∧ F[6,18](¬X)))`。`maneuver_crossing/head_on` = onset 起**累积 |Δθ_cum|≥Δlarge_turn=20°（sig_orientation_diff）∧ turning_to_starboard**（右转）；`maneuver_overtake` = 累积≥20°（**不要求 starboard**，可左可右）。违反 = 相遇全程未达成 maneuver_X，**每相遇** +1。
- 累积：`Δθ_cum += signed_modulo(θ[t−1]−θ[t], 2π)`（signed_modulo=sign·(|·|%|·|)；θ 减小=右转→正）；`change_course(Δθ,thr)=|Δθ|≥deg2rad(thr)`；`turning_to_starboard(onset_θ,cur_θ)`=净右转。
- 参数（yaml）：Δno_turn=10° / Δlarge_turn=20° / 2021 时间窗 [1,6]/[0,12]/[6,18] 步。

**实现方案 + 待定（step3(c)，诚实标注+可配置+step4 校准）**：`ViolationCounter`（逐 episode 有状态）直接用态势谓词（crossing/head_on/overtake/keep，已实现）+ 累积航向追踪：stand-on 每步 keep∧累积≥10° → +1；give-way 相遇 onset→解除全程未达 maneuver_X → +1。⚠️ **待定**：2024 未明示是否照搬 2021 精确 MTL 时间窗（F[0,12] 等）；本实现用**纯态势谓词 onset→解除全程累积**近似"per encounter no proper maneuver"（去精确步窗），step4 复现若对不上 Table III 再引精确时间窗校准。⚠️ 违规计数用**原始态势谓词**（非状态机 ρ，R_G3-6 直接基于谓词）；与安全盾 As(ρ) 解耦——它评估"裸策略违规多少"（基线 Base/RR），Safe agent 因有盾 → 违规 0。

**tracking_controller（Appendix-C p17，⭐ pymupdf 视觉读，文档"唯一待抽"已闭）**：李雅普诺夫位置跟踪。
- `Vω = 1 − ([cosθ,sinθ]·w_des^T)²`（=sin²航向误差）；`Va = 0.5·‖p_des−p_t‖²`；`w_des=(p_des−p_t)/‖p_des−p_t‖₂`。
- `ω = −λ1·Vω / [−2·([−sinθ,cosθ]·w_des^T)·([cosθ,sinθ]·w_des^T)]` — 即 `−λ1·Vω/V̇ω` 形式 → 闭环 V̇ω=−λ1·Vω 指数收敛。
- `a = −λ2·Va / [−(p_des−p_t)·[v·cosθ,v·sinθ]^T]`；`Vω>ΔVω` 时 a=0（只转向）。

# 二、Markgraf et al. 2026（投影方法论 + 开源实现）

**身份**：标题 **"Safe Reinforcement Learning using Action Projection: Safeguard the Policy or the Environment?"**，作者 Hannah Markgraf / Shambhuraj Sawant / Hanna Krasowski / Lukas Schäfer / Sébastien Gros / Matthias Althoff。**TMLR 03/2026**（已正式发表，OpenReview id=DDrGSEYxGU）。arXiv 2509.12833v2，2026-04-16。**开源代码：github.com/TUMcps/serl-sprl**。

> 一句话：投影式安全盾的**方法论**论文——理论 + 实证对比两种集成方式 SE-RL（盾当环境一部分）vs SP-RL（盾嵌进策略、可微）。给 TRB 提供"第三层 QP 投影"的现成路线 + 开源实现。

## ① SE-RL vs SP-RL（决定 TRB 怎么接 SAC）
- **SE-RL（安全环境 RL）**：投影盾当环境动力学一部分，RL 算法**完全不改**，critic 靠数据间接学盾的影响。
- **SP-RL（安全策略 RL）**：投影嵌进策略当**可微优化层**（cvxpylayers），梯度反传过盾，需算盾灵敏度（隐函数定理，附录 A.2）。
- 对 TRB 的指示：
  - **SE-RL 最省、最稳、不动 RL 算法** → ✅ 印证蓝图选 SE-RL。
  - SP-RL 算力贵 **3–12×** + 确定性策略下有收敛问题 → 蓝图把 SP-RL 列可选对照、时间紧可不做，合理。
  - **Theorem 1**：SE-RL 与 SP-RL 最优值函数相同。**Lemma 1/Corollary 1**：随机策略 + GAE（如 A2C）下两者**完全等价**（逐步更新一样）。

## ② ⭐ 理论明确排除 PPO/SAC（蓝图风险 §6/§9 确认）
原文 §5 原话："we limit our analysis to RL algorithms derived directly from the policy gradient theorem, as including methods with additional modifications (**e.g., PPO, Soft-Actor Critic (SAC)**) would require extensive theoretical derivations beyond the scope of this paper."
- 实验只用 **TD3**（确定性）+ **A2C**（随机），**无 PPO/SAC**。
- → **TRB 用 SAC 落在其严格理论覆盖外**，确认蓝图风险判断。应对（蓝图 §5/§6）：SAC 当工程求解器、卖点放"海事应用突破"非"投影理论性质"；要最干净理论可补 TD3 对照。

## ③ 投影 = 标准 QP（式12）= TRB 第三层
`Φ(x,u)=argmin_ũ ½‖ũ−u‖²₂  s.t. s(x,ũ)≤0`。安全动作集 `Uφ_x={u|s(x,u)≤0}`，**假设凸**（→ 投影唯一、确定性）；实践用凸内近似（多胞形/椭球/zonotope）。✅ = 蓝图第三层。只改不安全动作。

## ④ ⭐ 安全集用 zonotope（非多项式 zonotope）— 印证蓝图轻路线（附录 A.1）
- serl-sprl **用普通 zonotope**（Z=⟨c,B⟩，z=c+Bβ,|β|≤1），**不是** Krasowski [68] 指的多项式 zonotope。
- 两种约束：
  - **点在 zonotope 内**（安全集直接是 zonotope）：`Bu·ν=ũ−cu, ‖ν‖_∞≤1`（Kulmburg & Althoff 2021）= **最简单 = seeker 任务 = 蓝图 Route 1**。
  - **zonotope 含 zonotope**（不变集约束）：Ghasemi 2020 Lemma 1（B1=B2Γ, c2−c1=B2ω, ‖[Γ;ω]‖_∞≤1，等式+不等式）。
- → **蓝图 §6"zonotope+QP（轻）、避开多项式 zonotope（重）"完全正确**，serl-sprl 就是轻路线现成实现。

## ⑤ ⭐⭐ 档位 B 的现成工具 = RCI 集（最关键发现）
- 安全动作集可由**控制不变集**作安全状态集 Xφ：`x∈Xφ` 保证存在可行动作使约束对**所有未来时刻**成立 —— **这正是蓝图档位 B 的递归可行性/前向不变性**。
- serl-sprl 对 pendulum/quadrotor **用 RCI（robust control invariant 鲁棒控制不变）集**（**Schäfer et al. 2024**, "Scalable computation of robust control invariant sets of nonlinear systems", IEEE T-AC）；约束 = 下一步可达集 C⊆Xφ（含噪声 w∈W）的 zonotope 含入。
- → **TRB 档位 B（provably）的工具链已存在**：RCI 集 + zonotope 含入 + QP。难点不在工具，在**为两船动态 + COLREGS 方向约束构造 RCI/reach-avoid 集**（Phase 4 最不确定处，用你可达性背景）。

## ⑥ action aliasing 惩罚（蓝图 §3 确认）
- 现象：法锥内多个不安全动作投到同一安全动作 → 同 reward → 梯度丢。SE-RL 表现为"flat-lining critic"（依赖 critic 质量，不完美 critic 反缓解）；SP-RL 表现为"零梯度问题"（秩亏 Jacobian，恒存在，更糟）。
- **SE-RL 缓解 = 奖励加惩罚** `h=ξ(uφ,u)=w‖u−uφ‖²₂`（u 不安全时），w∈{0.1,0.5,1.0,2.0}。✅ = 蓝图 §3"奖励里加与投影修正量成正比的惩罚项"。
- SP-RL 缓解 = per-sample loss(PSL) 或本文新提 penalty critic(PenC)——SP-RL 专属，TRB 走 SE-RL 不需要。

## ⑦ seeker/navigation 任务 = 海事最近类比（附录 A.8）
- 点质量 seeker，最短路到目标 + 避障，**state-dependent 安全动作集**（非 RCI），任务与安全**不对齐**（障碍总在起点-目标间 → 最优安全动作落安全集边界）。
- ✅ = 蓝图 §4"Markgraf seeker 任务 = 点到安全集投影的简单情形"。**Phase 2 实现安全集时，seeker（附录 A.8.3）是最该照搬的模板**（本次未细读其构造，Phase 2 再读）。

## ⑧ 评估方法论（可迁移到 TRB / 也可比对 USV 那套）
interquartile mean（四分位均值）+ 95% 自助法置信区间，**7 training runs × 10 seeds**；统计用 Kruskal–Wallis + Dunn's test + Bonferroni 校正（p<0.05）。

## 对 TRB 的净结论
- **第三层 QP 投影 + 第二层 zonotope 安全集 = serl-sprl 现成**，走 SE-RL（轻、不改 SAC）。

- **档位 B** = RCI 工具链存在（Schäfer 2024），但要为两船 + COLREGS 自己构造不变集 = Phase 4 真难点。
- SAC 在其理论外 = 已知、可接受（卖点放应用突破）。
