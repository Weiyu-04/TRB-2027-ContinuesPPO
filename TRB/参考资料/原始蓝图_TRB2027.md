## 0. 一句话核心主张

可证明的 COLREGs 合规此前要么牺牲连续控制（Krasowski & Althoff 2024 用离散动作 + action masking 才能保证合规），要么牺牲合规保证（所有连续动作空间的海事避碰工作都退回了"软奖励"）。本文把投影式安全盾首次落到海事 COLREGs：用形式化状态机判定当前相遇态势与合规让路方向，在连续动作空间将"合规方向 + 动态无碰撞"刻画成一个 zonotope 安全动作集，再用二次规划投影把 SAC 输出的连续控制指令修正进该集合，从而在训练与部署全程实现可证明的规则合规，同时保留连续控制的平滑性与机动灵活性。

**"首次"声明的精确措辞（关键，措辞错一个词就被秒拒）**：

声明必须收窄到三重交集，一个限定词都不能少：

> *To the best of our knowledge, this is the first **provably (formally) COLREGs-compliant** collision-avoidance method that operates in a **continuous action space**.*

- **不能**写成"首次在连续动作空间做海事避碰"——连续动作空间的 COLREGs 避碰遍地都是（CyberShip II、Aeolus Ocean 连续 PPO、大量连续状态/动作工作），审稿人一秒甩五篇反例。
- 你打击的不是"连续避碰没人做"，而是：**连续避碰的工作全都只有软奖励、无硬保证；有可证明硬保证的（Krasowski 2024）又只能离散。** 这个区分必须在 Introduction 里一句话讲死。
- to the best of our knowledge 是方法论必需（无外部来源能证明"绝对没人做过"），投稿前再扫一轮 arXiv。
- **"provably" 这个词能否用，取决于你做到第 5 节的哪一档**：达到档位 B（不变集终端约束）才成立；只到档位 A（一步前瞻）必须去掉 provably，改用 "projection-based COLREGs-compliant action filtering"。投稿前按实际定稿。

## 1. 四象限定位（你在哪、别人在哪）

把"COLREGs 合规避碰"按两维切开：动作空间（离散 / 连续）× 合规保证强度（软奖励 / 可证明硬保证）。

| | 软奖励（无硬保证） | 可证明硬保证 |
|---|---|---|
| **离散动作** | 早期 DQN 类，大量 | **Krasowski & Althoff 2024**（action masking） |
| **连续动作** | **非常拥挤**（你那篇基线、大连海事多篇、PPO/SAC 各种 2023–2025） | **← 本文（基本空白）** |

支撑"这一格空着"的三重旁证：
1. Krasowski 2024 在论文里把"连续动作空间"列为未来工作，并指出可用动作投影来做。
2. 实现这条路的通用方法存在且成熟（Gros 2020；Kochdumper/Krasowski 2023；Markgraf 2026），明确支持连续系统、输入约束、动态障碍 —— 但从未被实例化到海事 COLREGs。
3. 2025-09 最新海事安全 RL 论文（arXiv 2509.01838）仍卡在离散动作空间，并把"对齐连续控制"列为自身未解决局限。

## 3. 方法三层结构

```
观测 → [SAC 策略] → 连续动作 u
                          │
                          ▼
   ┌─────────────────────────────────────────┐
   │ 第一层：相遇态势与合规方向判定（复用状态机） │
   │  判定 keep / head-on / crossing / overtake │
   │  及本船是让路方还是直航方、合规让路方向     │
   └─────────────────────────────────────────┘
                          │
                          ▼
   ┌─────────────────────────────────────────┐
   │ 第二层：连续安全动作集刻画（zonotope）      │
   │  "合规方向 + 与他船预测占据集无碰撞"        │
   │  → 一个凸的 zonotope 安全动作集 U^φ_x       │
   └─────────────────────────────────────────┘
                          │
                          ▼
   ┌─────────────────────────────────────────┐
   │ 第三层：QP 投影                            │
   │  若 u ∈ U^φ_x 直接执行；否则投影到最近点    │
   │  min ½‖ũ−u‖²  s.t. ũ ∈ U^φ_x              │
   └─────────────────────────────────────────┘
                          │
                          ▼
                  安全动作 u^φ → 环境
```

- **关键工程事实（复查确认）**：CommonOcean 的底层动力学（偏航受限模型，控制输入为连续的加速度 + 转艏率）**本身就是连续的**。Krasowski 是为了让离散 action masking 可行，才人为把动作空间离散成 49 个动作。因此你**不需要换环境、不需要改动力学**——只需解除她的人为离散化，直接在连续控制输入 `u = [a, ω]` 上做投影。同一环境、同一动力学、同一状态机判定，只把"离散枚举 + 屏蔽"换成"连续投影"。这让你与她的对比无可挑剔地干净。
- **集成方式**：优先用 SE-RL（安全盾当环境一部分，SAC 主体不改，最省事、最稳）。SP-RL（投影嵌进策略、可微层）作为可选对照，时间紧可不做。
- **合规靠投影硬保证，不靠奖励**。奖励只保留导航目标项（到达、平滑、速度区间）。
- **action aliasing 缓解**：按 Markgraf 的结论，在奖励里加一个与"投影修正量"成正比的惩罚项（SE-RL 的标准做法），缓解效率损失。
- **第二层的具体数学构造**见本文档第 12 节（连续 COLREGs 安全动作集的构造初步思路）——这是整个方案最硬的一环，已单独展开。

## 4. 实验设计

**环境**：直接用 CommonOcean（Krasowski 团队开源），保证与基线严格可比。

**对比对象**（前三个沿用 Krasowski 设置，第四个是你的）：

| 智能体 | 合规机制 | 动作空间 | 角色 |
|---|---|---|---|
| Baseline | 无 | 连续 | 下限参照 |
| Rule-reward | 软奖励 | 连续 | 代表现有 SOTA（含你那篇基线类） |
| Discrete-safe | action masking | 离散 | Krasowski 2024，可证明合规对照 |
| **Continuous-safe (本文)** | **QP 投影** | **连续** | 你的方法 |

**场景**：对遇 / 交叉（左、右）/ 追越，叠加他船数量（先做单他船，把双他船作为扩展）。

**核心指标表（双栏对照，沿用 Krasowski 结构）**：

| | 效率组 | 安全组 |
|---|---|---|
| 指标 | 到达率、轨迹长度、控制平滑度 | COLREGs 违规率、碰撞率、安全介入率 |

**钱图（money plot）**：效率-合规权衡图。要展示的核心结论 —— 连续投影达到与离散 masking 同样的零违规零碰撞，但效率指标更优、控制更平滑（即把 Krasowski Table III 里掉的 4–5% 到达率代价压低）。

## 5. 贡献点清单（写进 Introduction）

1. 首个在连续动作空间下实现可证明 COLREGs 合规的安全 RL 避碰框架（to the best of our knowledge）。
2. 将带方向与时序的 COLREGs 让路逻辑刻画为连续 zonotope 安全动作集 + QP 投影的具体构造。
3. 在 CommonOcean 上与离散 masking、软奖励、无约束基线的系统对比，量化"连续投影 vs 离散屏蔽"的效率-合规权衡，验证连续投影在保持零违规的同时降低安全代价。

## 12. 第二层数学构造：连续 COLREGs 安全动作集（初步思路）


> 目标：把"当前相遇态势下，合规让路方向 + 与他船无碰撞"翻译成连续控制输入上的一个**凸 zonotope 安全动作集** U^φ_x，使得 SAC 输出的连续动作可投影进去。这是方案最硬的一环，先给出可施工的初步构造。

---

## 0. 衔接：动力学与符号（沿用 Krasowski 偏航受限模型）

本船状态 `s = [p_x, p_y, θ, v]`（位置、艏向、纵向速度）。连续控制输入：

```
u = [a, ω],   a = 纵向加速度,  ω = 转艏率
ṗ_x = v·cosθ,  ṗ_y = v·sinθ,  θ̇ = ω,  v̇ = a
```

动作受限：`a ∈ [a_min, a_max]`，`ω ∈ [ω_min, ω_max]`。
这个箱式约束本身就是一个 2D zonotope（记为 `U`），中心 + 两个轴对齐生成元。我们要做的是在 `U` 内部再"切"出合规且安全的子集 `U^φ_x`。

## 1. 总体思路：三个约束的交集

在某个时刻、某个相遇态势下，安全动作集是三类约束的交：

```
U^φ_x  =  U_box  ∩  U_colregs(态势)  ∩  U_collision-free(他船预测)
```

- `U_box`：物理动作限幅（已是 zonotope）。
- `U_colregs`：COLREGs 让路方向约束（把"应向右转/应保向"翻译成对 ω 的半空间约束）。
- `U_collision-free`：一步前瞻无碰撞（把"下一步本船可达占据集与他船预测占据集不相交"翻译成对 u 的约束）。

三者都设计成**线性/半空间约束**，交集是凸多胞形；再用 zonotope（或受约束 zonotope）内近似，得到可投影的凸集。投影即标准 QP（见蓝图第 3 层）。

---

## 2. 第一类：COLREGs 方向约束 U_colregs

复用 Krasowski 状态机判定当前态势属于 {no-conflict, keep(直航), head-on, crossing, overtake} 中哪一个，以及合规让路方向。把每种态势映射成对控制输入的线性约束：

| 态势 | COLREGs 要求 | 对 u 的约束（初步） |
|---|---|---|
| no-conflict (ρ0) | 无 | 无额外约束，U_colregs = U_box |
| keep / 直航 (ρ1) | 保向保速 | ω ≈ 0 且 a ≈ 0：`|ω| ≤ ε_ω, |a| ≤ ε_a`（窄带，近似"保持"） |
| head-on (ρ4) | 右转避让 | `ω ≤ -ω_turn`（约定 ω<0 为右转）即要求足够的右转率 |
| crossing 让路 (ρ3) | 右转避让 | 同 head-on：`ω ≤ -ω_turn` |
| overtake 让路 (ρ5/给定方向) | 按几何定的左/右 | 据状态机给出的方向：`ω ≤ -ω_turn` 或 `ω ≥ +ω_turn` |

要点：
- 每条都是对 `ω`（必要时含 `a`）的**半空间约束**，与 `U_box` 求交仍是凸多胞形。
- "足够大转向"对应 Krasowski 的 `Δ_large_turn`（让他船能辨识让路意图）——这里转成"转艏率下限"而非离散搜索。
- "保向"带不写成硬等式（否则集合退化为线段、投影病态），用**窄带 ε** 近似，保留可投影的二维体积。

## 3. 第二类：一步前瞻无碰撞 U_collision-free

这是用到你可达性背景的核心。思路：用集合表示保证"采用动作 u 后的本船下一步占据集，与他船在该时段的预测占据集不相交"。

**3.1 他船预测占据集（保守、不需其控制输入）**
沿用 Krasowski 的 set-based prediction：他船用点质量模型 + 速度/加速度上界做可达集，投影到位置域并按船体尺寸膨胀，得到时间区间占据集 `O_obs(τ)`（一个随 τ 增长的 zonotope/多胞形序列）。规则态势下可假设他船保向保速（更紧），紧急态势下用全可达集（更保守）。

**3.2 本船一步可达占据集随 u 的依赖**
给定当前状态 s 和动作 `u=[a,ω]`，前向一个决策步 Δt 得到本船位姿，再按船体尺寸膨胀为占据集 `O_ego(s, u, τ)`。难点：`O_ego` 对 `u` 是**非线性**的（含 cos/sin）。

**3.3 分离超平面构造（让约束变成对 u 的线性约束）— 复审后修订**

> 复审修订说明：初版用"对一步前向映射做泰勒线性化"，会引入线性化误差、保守性来源不干净。复审检索（避碰 CBF / MPC 文献）确认更标准、更稳的做法是**分离超平面定理**：把"本船占据集与他船占据集不相交"精确表达为"存在一个分离超平面"，并用双层结构在求解前先把超平面算好、再作为参数嵌入。改用此法。

两步：

**(i) 先求分离超平面（在投影 QP 之前，把超平面当参数算好）**
给定当前本船标称下一步占据集 `O_ego^nom` 与他船预测占据集 `O_obs(τ)`，求一个超平面 `{z : nᵀz = b}` 把两者分开：本船一侧 `nᵀz ≤ b − d_safe`，他船一侧 `nᵀz ≥ b`。两个凸集（zonotope/椭球/多胞形）之间的分离超平面可由一个小凸可行性问题（或对偶 QP）解出 `(n, b)`。这一步把"超平面"从优化变量里消掉，使下一步对 u 保持线性。

**(ii) 再把"本船下一步须留在安全侧"写成对 u 的线性约束**
本船下一步位置 `p_ego(s,u)` 关于 u 仍含 cos/sin 非线性，但只需保证 `nᵀ p_ego(s,u) ≤ b − d_safe`。对这一**标量约束**关于 u 做一阶展开即可（只需位置对 u 的雅可比，远比对整个占据集线性化简单、误差也只体现在这一个标量上）：

```
nᵀ p_ego(s,u) ≈ nᵀ p_ego(s,u₀) + nᵀ J_p (u − u₀) ≤ b − d_safe
```

`J_p = ∂p_ego/∂u |_(s,u₀)`。整理后是对 u 的一条线性不等式。每个相关他船 / 每个前瞻子时刻 τ 各贡献一条，合起来与 U_box、U_colregs 求交仍是凸多胞形。

要点：保守性现在只来自三处可控来源——分离超平面的安全裕度 `d_safe`、标量约束的一阶展开余项、前瞻时刻的离散采样——比"整集线性化"清晰得多，便于在论文里诚实界定保证范围。

**3.4 保守性补偿**
(a) 安全裕度 `d_safe` 取大（Krasowski 用 2~3 倍船长）；(b) 用区间/zonotope 把 (ii) 中标量一阶展开的余项 over-approximate 进约束（含入余项上界即可恢复硬保证）；(c) 决策步 Δt 内多采样几个子时刻 τ，对每个 τ 都加分离约束。宁可保守——保守不破坏"可证明无碰撞"，只略损效率，正是蓝图风险表里的既定取舍。

> **诚实标注（同行复审指出）**：(b) 没有看起来那么轻。要算余项上界，需对 `nᵀp_ego(s,u)` 关于 u 做**二阶项的全局区间界**（含 cos/sin，Lagrange 余项里有 Hessian）。在 u 的整个箱式域上做这个界会**很保守**；要不保守就得细分区间，**工程量上来**。实质是"要么很松、要么很烦"的二选一。初版务实做法：先用大 `d_safe`（手段 a）兜住，把 (b) 的严格余项界作为档位 B 严密化时再补的步骤，不要在第 1–2 周就卡在这里。

## 4. 求交、内近似与投影

```
P  =  U_box ∩ U_colregs ∩ U_collision-free      （凸多胞形，H-表示）
U^φ_x  =  zonotope/约束zonotope 内近似(P)        （可选，便于统一表示与含 0 判断）
投影:  u^φ = argmin_{ũ ∈ U^φ_x} ½‖ũ − u‖²        （标准 QP）
```

两条实现路线，二选一：
- **路线 1（更简单，建议先做）**：直接在 H-表示多胞形 `P` 上投影——这就是一个带线性不等式约束的 QP，cvxpy / OSQP 直接解，不必显式转 zonotope。Markgraf 论文的 seeker 任务正是这种"点到安全集投影"的简单情形。
- **路线 2（与文献表示统一）**：把 `P` 内近似为 zonotope，用 zonotope 含入约束（Markgraf 附录 A.1 的公式）。表述更"形式化"，但工程更重。**时间紧先走路线 1。**

**可行性兜底（必须处理，且必须验证）**：若某时刻 `P = ∅`（合规约束与无碰撞约束冲突，多见于 >2 船或紧急局面）——这正是 Krasowski 用"紧急机动规则 R1"覆盖的情形。初版处理：当 `P=∅` 时切换到"碰撞风险最小化"的紧急动作（放松 COLREGs 方向约束、只保无碰撞），并在论文里明确这对应紧急规则、且 COLREGs 对此类冲突本无明确规范。

> **同行复审指出的洞**：单他船场景里 `P=∅` 几乎不发生，兜底机制根本不会被触发，"我们处理了可行性兜底"就成了纸上谈兵——而审稿人恰恰想看冲突情形下你怎么办。**因此第 4 周必须构造几个人工紧急冲突场景**（例如两条他船夹击、或他船不守规突然逼近）来真正触发并验证兜底逻辑，哪怕不做完整双他船统计。这把"双他船"从原来的"可选扩展"提升为"至少要有触发兜底的冲突算例"。

## 5. 可证明性论证的落点（写作时怎么说"provable"）— 复审后重写

> **复审发现（致命级，必须正视）**：一步前瞻 ≠ 可证明安全。形式化安全文献明确，全程安全需要**递归可行性 / 前向不变性**（初始可行集 = 最大正不变集）。从最大不变安全集之外出发，没有静态策略能以概率 1 保证全程安全。也就是说：只保证"这一步投影后不撞"，智能体可能被一步步逼进"下一步怎么动都要撞/违规"的死角，安全集到那一刻才发现为空。**若声明 provable 却只做一步前瞻，会被审稿人（很可能正是 Krasowski 圈子的人）一击毙命。** 这是 Krasowski 为何要做紧急控制器 + 集合可达性兜底的根本原因。

**因此"provable"这个词的强度，取决于你做到哪一档。三档按时间/野心排：**

| 档位 | 你做了什么 | 能诚实声明的强度 | 工作量 |
|---|---|---|---|
| **A（最小，保底）** | 一步前瞻投影 + `P=∅` 时切紧急动作（风险最小化，不保不变性） | **不能**用 "provably safe"。只能写 "rule-compliant by construction at each step" / "one-step COLREGs-compliant action filtering"。安全是**经验性**的（实验零碰撞），不是定理 | 低，1 个月稳妥 |
| **B（推荐目标）** | 预计算一个**鲁棒控制不变集 / reach-avoid 集**作为终端约束：投影时不仅要求"这一步安全"，还要求"落点仍在不变集内（存在后续合规逃逸策略）" | **可以**用 "provably COLREGs-compliant and collision-free"，但限定在两船、已知动力学、所采用的不变集近似下。这就是真正的贡献级声明 | 中高，是方案最硬部分；你的可达性背景正用在这里 |
| **C（满分，超纲）** | 完整连续时间前向不变性 + 形式化验证 | 最强声明 | 一个月内不现实，明确列 future work |

**务实建议**：把 **B 作为目标**，**A 作为保底**。即第 1–2 周先把 A 跑通（保证有结果可投），第 3–4 周尽力做到 B；若 B 的不变集在时间内做不稳，**就老实退回 A 的措辞**——宁可声明弱一点为真，绝不声明 provable 却只有一步前瞻。这个取舍直接决定标题里能否出现 "provably"。

> **⚠ SAC 与档位 B 的真实张力（同行复审指出，必须正视，不能再用"求解器无关"糊过去）**：
> "保证由投影提供、与求解器无关"这句话**只在档位 A 成立**——档位 A 的保证就是单步静态投影，确实独立于策略。但到**档位 B 就不成立**：递归可行性要求"落点仍在不变集内、存在后续逃逸策略"，这隐含了对**闭环策略行为**的要求，不是纯静态投影能独立保证的。**你越往 provable 走，SAC 这个选择越站不住。** 应对方向（按代价排）：(i) 档位 B 下把不变集的终端约束也做进投影的约束集（让"落点∈不变集"成为投影硬约束之一），使保证尽量回到静态投影层面；(ii) 接受"档位 B 的保证是相对于不变集近似 + 该终端约束而言"的限定，并在 Limitations 诚实写明；(iii) 若想要最干净的理论,改用 Markgraf 理论覆盖的算法(如 TD3)做档位 B 的版本、SAC 仅作档位 A/工程对照——但这增加工作量。一个月内现实选 (i)+(ii)。

**无论 A 还是 B，写作时的诚实边界（照抄进 Limitations）**：
- 保证相对于所采用的近似（分离超平面 + 标量一阶展开余项含入 + 离散前瞻时刻）与集合表示而言，非连续时间全局绝对保证；
- 仅限两船：COLREGs 对 >2 船本无明确规范（Krasowski 亦如此论证），多船列 future work；
- 这与 Krasowski 的诚实程度对齐（她也把连续空间、多船、参数可满足性证明列为未来工作）。

**对"首次"声明的反向校准**：标题/摘要的 "provably COLREGs-compliant ... in continuous action space" 只有在你达到**档位 B** 时才完全成立。若最终只到 A，把声明改为 "continuous-action COLREGs-compliant collision avoidance via projection-based action filtering"，去掉 provably，仍是有效且诚实的贡献（首个连续动作的投影式 COLREGs 滤波 + 系统对比），只是新颖性叙事从"理论保证"转向"方法 + 实证"。**这条必须在投稿前根据实际做到的程度定稿。**

## 6. 初版实现的"最小可证明"目标（第 1–2 周验证用）

为降风险，先做能成立的最小版本（= 档位 A），再加码到 B：
1. 只做**单他船**、**对遇(head-on)**一种态势。
2. `U_colregs`：`ω ≤ -ω_turn`（右转）。
3. `U_collision-free`：单一前瞻时刻 τ=Δt，**分离超平面 + 标量一阶展开 + 大安全裕度**（见 3.3 修订版），得一条线性约束。
4. 路线 1 的 QP 投影。
5. 验证：SAC 输出被投影后，整段轨迹零违规、零碰撞、能到达目标（此时是**经验性**零碰撞，对应档位 A 措辞）。

## 14. COLREGs 违规率判定标准（投稿前必须锁死，与 Krasowski 同口径）

> 同行复审指出：通篇拿"零违规"当卖点，却没定义违规率怎么算。COLREGs 量化无统一标准；若口径与 Krasowski 不一致，"我们和离散 masking 同样零违规"的对比直接不成立。此节把指标锁死，**第 1 周完成、与被比较方法严格同口径**。

**对齐基准**：直接采用 Krasowski & Althoff 2024 论文的违规计数口径（她在第 VII 节定义），逐条对齐，不自创：
- 直航(stand-on)态势：每一个本应保向却发生偏离的时间步计一次违规；
- give-way 态势（对遇/交叉/追越）：每一个本该让路却未采取合规避让机动的相遇计一次违规。

**需要在第 1 周明确写死的判定阈值（全部取 Krasowski 同值，列进方法节一张参数表）**：
- 相遇态势分类的扇区角阈值（front/left/right/behind、head-on 角带 Δ_head-on）；
- "明显转向"幅度阈值 Δ_large_turn（让路是否被判定为"采取了合规机动"）；
- "保向"容许带 Δ_no_turn（直航是否被判定为"保持"）；
- 碰撞危险判定 collision_possible 的 CPA / 时域 t_horizon 参数；
- 让路方向正确性（对遇/交叉须右转）。

**写作时的关键声明**：明确写"我们采用与 Krasowski & Althoff (2024) 相同的 COLREGs 违规判定与参数，以保证 Discrete-safe 与 Continuous-safe 的违规率严格可比"。这一句直接堵掉"指标不可比"的致命质疑。

**注意**：你的方法是连续动作，Krasowski 是离散；违规率的**判定**口径要同，但要留意"明显转向"在连续控制下如何换算（她的 Δ_large_turn 对应离散转向动作，你需把它表达为连续轨迹上的累计航向变化阈值——换算关系要在方法节交代清楚，否则又是一个可比性漏洞）。
